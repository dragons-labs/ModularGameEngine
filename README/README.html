ABOUT

Modular Game Engine is game engine written in C++20 and using: Ogre 3D, OIS, CEGUI, Python, Boost, OpenAL (via OgreOggSound), Bullet (via BtOgre).

Modular Game Engine is free and open source software available under permissive license (MIT License). It contains some pieces of code from other projects developed under MIT License, Zlib license, Public Domain (un)licence. See full copyright info in COPYRIGHT.

Main engine feature:

  • small and simple
    • direct use of graphics, physics, etc libraries
    • avoids create own implementations of classes that are contained in the used libraries
    • (if you know used libraries, you don't have to waste time learning new APIs for them)
  • modular designed:
    • game world, logic, missions, GUI elements and any game objects are "modules" / "plugins"
    • actions, dialogs, etc are python script
    • modular design of engine code
    • GUI appearance is setting in xml files
    • Python3 script system and API
  • XML config and saves (mostly cases used the same syntax)
    • allows easy modification game behavior
    • allows use the same methods for save-restore and config-load
  • decent graphics (via OGRE) and sound (via OpenAL) with 2D and 3D support
  • decent physics (via Bullet)
  • support for:
    • actors, objects selection
    • actions (context) menu
    • different action target: world point, objects list, area, polygonal chain
    • actors action queued
    • dialog menu with audio support
    • interactive textures for: CEGUI, web browser, VNC (probably this is first computer game engine with builtin VNC client), ...
  • flexible system of camera mangament with:
    • simple support for RTS-style camera movement and object attached camera
    • support for multiple view (view from other camera in sub-window)
  • object (actor) properties system (to do something we need object with correct feature, not specified object)
  • game speed regulation witch active and full pause option
  • desing not as single library, but as framework with generic engine modules, application initialization modules and application logic modules

BUILD AND RUN

  1. Install dependencies - see README/BUILD.deps for info. Order is important.
  2. Create build dir and enter to it (all next commands should be run in this directory):
    mkdir build; cd build
  3. Run cmake with CMAKE_CXX_COMPILER set to selected compiler and with path to SRC directory:
    cmake -D CMAKE_CXX_COMPILER=clang++ ..
  4. Run make:
    make -j 5
  5. Run game:
    ./Game
